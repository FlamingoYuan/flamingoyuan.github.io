使用 GitHub Pages 预览 HTML

什么是 GitHub Pages

GitHub Pages 是 GitHub 提供的一个网页寄存服务。可以用于存放静态网页，包括博客、项目文档甚至整本书。一般 GitHub Pages 的网站使用 github.io 的子域名，但是用户也可以使用第三方域名。

GitHub Pages

如何使用 GitHub Pages

假设我们已经上传了一个本地仓库到 GitHub 上，我们在 Github 中打开这个仓库，仓库中已经包含 index.html 文件。

如果直接点击 index.html，我们没法看到网页的预览，而只能看到网页源代码。

接下来我们开启 GitHub Pages 功能，点击 Settings。

往下翻找到 GitHub Pages，选择 master branch，这时 GitHub Pages 就已经开启了。

可以看到 GitHub Pages 提供了一个链接

复制这个链接，并在后面加上 index.html (flamingoyuan.top/blog/index.html) 即可预览我们的网页，实际上，即使后面不加 index.html 也可以成功预览它, GitHub Pages 默认会访问 index.html, 如果要预览其他网页，则必须加上相应的路径。

当然如果你也可以自定义域名，只需要在 custom domin 中输入域名，点击 save 就可以。前提条件是你已经购买了域名并配置好了域名解析。具体教程可参考：
